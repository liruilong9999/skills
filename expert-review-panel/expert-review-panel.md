# 专家组评审 Skill（强制规则）

## Goal
模拟由至少3名不同角色专家组成的评审组，对输入的需求文档或设计文档进行全面、专业、客观的评审。发现问题、提出改进建议，并生成结构化的评审报告，存放至指定路径。

---

## 1. 输入要求
- 输入文档类型：必须明确是「需求文档」还是「设计文档」
- 输入文件：完整的结构化需求文档（PRD_vX.md）或软件设计文档（SDD_vX.md）
- 评审版本：如 v1、v2 等

---

## 2. 专家组组成（强制至少3名专家）
每次评审固定组建以下3名核心专家（可根据文档类型适当增加，这里需要创建子智能体，防止互相干扰）：

1. **架构师专家**（Architect Expert）  
   - 专注系统架构合理性、分层设计、扩展性、技术选型、可维护性

2. **领域/业务分析师专家**（Business Analyst Expert）  
   - 专注需求完整性、业务逻辑一致性、用户场景覆盖、边界条件、traceability

3. **开发与质量专家**（Development & QA Expert）  
   - 专注可实现性、性能指标落地、异常处理、测试可行性、代码开发友好度、安全细节

---

## 3. 输出文件存放规范（强制）
- 需求文档评审报告路径：`/docs/评审/需求_vX.md`  
  示例：`/docs/评审/需求_v1.md`
- 设计文档评审报告路径：`/docs/评审/设计_vX.md`  
  示例：`/docs/评审/设计_v1.md`

---

## 4. 评审报告模板（强制严格遵循以下结构）

```txt
# 【需求/设计】文档评审报告_vX

## 文档信息
| 项目 | 内容 |
|------|------|
| 评审文档类型 | 需求文档 / 软件设计文档 |
| 评审文档版本 | vX |
| 评审日期 | YYYY-MM-DD |
| 评审专家组 | 架构师专家、业务分析师专家、开发与质量专家 |

## 专家组评审总结

主要问题：
建议版本：继续使用 vX / 修订后进入 vX+1

## 1. 架构师专家评审意见
- 总体架构评价
- 分层与模块划分合理性
- 扩展性与可维护性评估
- 技术选型与性能设计建议
- 严重问题（High）
- 中等问题（Medium）
- 建议改进点（Low）

## 2. 业务分析师专家评审意见
- 需求/设计与原始业务目标的一致性
- 功能覆盖完整性与边界定义清晰度
- 用户角色与场景覆盖情况
- traceability（需求与设计之间的追溯性）
- 模糊点或遗漏的业务逻辑
- 严重问题（High）
- 中等问题（Medium）
- 建议改进点（Low）

## 3. 开发与质量专家评审意见
- 可实现性与工程落地难度评估
- 性能指标是否可量化落地
- 异常处理、容错机制完整性
- 测试与验收标准可操作性
- 安全设计细粒度
- 代码开发友好度（模块独立性、接口清晰度）
- 严重问题（High）
- 中等问题（Medium）
- 建议改进点（Low）

## 5. 改进建议与行动项
- 建议修改重点
- 预计修改后可进入下一阶段（设计 / 开发）
- 推荐下一次评审时机

## 6. 评审结论
- 是否推荐进入下一阶段：是 / 否 / 有条件通过
- 最终建议：直接通过 / 修订后通过 / 需要重大重构

## 附录
- 评审专家签名（模拟）：架构师专家 / 业务分析师专家 / 开发与质量专家
```

## 5.评审执行规则（强制）

- 每位专家必须从自身专业视角独立给出意见，不得泛化。
- 必须明确区分 High（严重，必须修改）、Medium（重要，建议修改）、Low（优化，可选） 三级问题。
- 所有问题必须可操作、可追溯（指明具体章节或编号）。
- 评审意见必须客观、中立、专业，避免空洞表扬。
- 如果文档存在明显遗漏或重大风险，必须在「综合问题清单」中重点标注。
  生成评审报告后，在报告末尾添加提示：“如需根据评审意见修订文档，请提供修订后的新版本进行下一轮评审。”

## 6.Enforcement（强制）

- 专家组必须至少包含上述3名专家，不得少于3人。
- 输出文件必须严格按指定路径和命名规则存放（需求_vX.md 或 设计_vX.md）。
- 评审报告必须结构完整，不得缺失任何章节。
- 语言严谨、专业，使用中文描述。
- 本 Skill 可同时支持对需求文档和设计文档的评审，只需在输入时明确文档类型。

## 使用说明

- 输入时请提供完整的需求文档或设计文档内容，并明确说明“评审需求文档 v1” 或 “评审设计文档 v1”
- 本 Skill 将自动生成评审报告并存放到对应路径：
  - 需求评审 → /docs/评审/需求_v1.md_
  - _设计评审 → /docs/评审/设计_v1.md

- 可多次调用本 Skill 对修订后的版本进行多轮评审。
