# Tech Research Expert Team

## 1. Purpose

该 Skill 用于处理以下高复杂度工程与研究任务：

### 核心场景
- 技术选型（A vs B vs C）
- 技术路线调研（工业/论文/SOTA）
- 系统架构设计
- 可行性分析
- 技术方案对比
- 技术方案评审
- 技术研究报告生成

---

## 2. When to use this skill

必须触发条件：

- 用户要求“技术调研 / research / survey”
- 用户要求“技术选型 / 技术路线分析”
- 用户要求“方案对比 / 架构设计”
- 用户询问“如何实现某系统”
- 用户需要论文+工业实现综合分析

---

## 3. Output Requirement (强制)

### 📌 必须输出文档

所有结果必须生成 Markdown 文档，并写入：

```txt
/docs/技术路线分析/{主题}调研.md
或
/docs/技术路线分析/{主题}分析.md
```
### 文件命名规则

| 类型   | 文件名     |
| ------ | ---------- |
| 调研类 | xxx调研.md |
| 分析类 | xxx分析.md |

示例：

- /docs/技术路线分析/语音增强调研.md
- /docs/技术路线分析/目标检测算法分析.md

## 4.Multi-Agent System

### 4.1 Survey Engineer（调研工程师）

- 搜索工业实现（GitHub / docs / blog）
- 收集主流工程方案
- 分类技术路线

输出：

- 工业方案列表

### 4.2 Paper Analyst（论文分析专家）

- 检索论文（arXiv / IEEE / CV领域会议）
- 提取方法与创新点
- 判断是否工程化

输出：

- SOTA方法总结

### 4.3 System Architect（架构专家）

- 系统拆解
- 模块设计
- 数据流分析

输出：

- 架构设计

### 4.4 Feasibility Judge（可行性评估）

评估维度：

- 性能
- 成本
- 复杂度
- 成熟度
- 可扩展性
- 风险

输出：

- 评分矩阵
- 风险分析

## 5.Workflow

### Step 1：问题解析

- 明确目标
- 提取约束条件

### Step 2：技术调研（必须Web搜索）

必须包含：

- 工业实现（企业 / GitHub）
- 论文（近3~5年）
- survey / benchmark

### Step 3：方案分类

必须至少三类：

- Baseline（传统方案）
- Industry Standard（工业主流）
- SOTA / Research（前沿方案）

### Step 4：对比分析

| 维度     | 说明        |
| -------- | ----------- |
| 性能     | 延迟 / 吞吐 |
| 成本     | 资源消耗    |
| 复杂度   | 实现难度    |
| 成熟度   | 工业落地    |
| 可扩展性 | 扩展能力    |
| 风险     | 稳定性      |

### Step 5：输出决策

必须包含：

- 推荐方案（1个）
- 备选方案（1~2个）
- 不推荐方案（原因）
