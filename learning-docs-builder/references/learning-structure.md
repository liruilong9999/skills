# 学习资料结构参考

本参考文件用于帮助你为不同主题设计一套“从入门到精通”的项目内学习资料。

只有在你需要决定文档拆分方式、每篇文档应该覆盖什么内容、或者如何平衡概念与实践时，再读取本文件。

## 1. 先判断主题类型

优先判断主题属于哪一类，再决定资料结构：

### A. 概念型主题

示例：

- OPC UA
- Kafka
- CAP 理论
- 微服务治理

推荐加强：

- 背景与问题域
- 核心概念关系
- 协议或机制原理
- 常见误区
- 工程落地约束

### B. 工具型主题

示例：

- Docker
- Git
- Prometheus
- Playwright

推荐加强：

- 安装与环境搭建
- 常用命令或基本操作
- 典型工作流
- 常见故障排查
- 进阶配置与最佳实践

### C. 框架型主题

示例：

- Vue 3
- React
- FastAPI
- Spring Boot

推荐加强：

- 核心抽象
- 开发模式
- 生命周期或运行机制
- 项目结构建议
- 性能与工程化

### D. 源码/架构型主题

示例：

- 某个开源项目的源码阅读
- 某个公司内部系统架构

推荐加强：

- 模块划分
- 主流程
- 关键数据流
- 扩展点
- 调试与二次开发入口

## 2. 默认文档蓝图

如果用户没有指定更细粒度结构，默认生成以下内容：

### 00-学习导航.md

必须包含：

- 学习目标
- 适合人群
- 阅读顺序
- 完整文档地图
- 建议学习周期
- 快速开始路径

### 01-入门总览.md

必须回答：

- 这个主题解决什么问题
- 为什么值得学
- 与相邻技术相比有什么差异
- 新手先建立什么直觉

### 02-核心概念.md

必须包含：

- 核心术语
- 关键对象或组件
- 概念关系图
- 容易混淆的对比项

### 03-基础实践.md

必须包含：

- 环境搭建
- 第一个最小示例
- 基本操作流程
- 新手最容易踩的坑

### 04-关键机制与常用模式.md

必须包含：

- 常见设计模式或使用模式
- 关键机制的内部逻辑
- 不同场景下的选型建议

### 05-进阶专题.md

可根据主题选择：

- 性能优化
- 安全机制
- 可扩展性
- 分布式特性
- 高可用设计

### 06-工程实践与案例.md

必须尽量贴近项目：

- 当前项目如何使用该主题
- 在真实系统中如何落地
- 配置、监控、排障、发布的注意点

### 07-常见问题与误区.md

必须包含：

- 常见误解
- 错误用法
- 排障思路
- 决策边界

### 08-精通路线图.md

必须包含：

- 从会用到精通的阶段划分
- 深入阅读建议
- 练习建议
- 可验证的能力里程碑

## 3. 可选增强文档

当主题较复杂时，可以补充以下文件：

### 09-练习题与参考答案.md

适合需要系统训练的主题，包含：

- 基础题
- 场景题
- 综合题
- 简要参考答案或思路

### 10-术语表.md

适合术语密集的主题，包含：

- 英文术语
- 中文解释
- 一句话定义
- 常见上下文

### 11-源码阅读指南.md

适合框架、协议栈、SDK 或大型系统，包含：

- 阅读入口
- 推荐阅读顺序
- 关键模块
- 调试断点建议

### 12-面试题与讨论题.md

适合需要知识检验或团队分享的主题，包含：

- 概念题
- 场景题
- 设计题
- 反例讨论题

## 4. 每篇文档的推荐写法

尽量采用以下顺序：

1. 本篇解决的问题
2. 核心概念或机制
3. 示例
4. 工程实践提示
5. 常见错误
6. 小结
7. 自测题
8. 下一步

## 5. 质量检查清单

输出前逐项检查：

- 文档顺序是否自然，是否真能从零开始学
- 是否同时覆盖“是什么、为什么、怎么做、什么时候别这样做”
- 是否至少有一个最小实践示例
- 是否把术语首次出现写成“英文(中文解释)”格式
- 是否避免空洞表述与未完成占位语
- 是否与当前项目已有文档结构兼容
